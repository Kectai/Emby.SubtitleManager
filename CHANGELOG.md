# Changelog

## Unreleased

## v1.3.4 - 2026-05-10

### 安全

- 将插件 ID 从模板占位 GUID 更换为一次性生成的稳定 GUID。
- 修复并发上传同名字幕时失败请求可能误删成功请求文件的竞态问题。
- 非管理员访问插件 API 时改为权限错误响应，不再返回空业务结果。
- 删除字幕时优先使用字幕流索引定位，保留路径参数作为兼容后备，减少前端对内部文件路径的依赖。

### 修复

- 修复普通物理文件夹（例如 `Specials/Season N`）被 extras 补充查询覆盖递归缓存上下文后，嵌套子文件夹无法继续展开影片的问题。
- 切换媒体库或重新搜索时同步清理前端选中项、展开状态和媒体缓存，避免跨媒体库状态污染。

### 优化

- 媒体库列表改用 Emby 用户视图接口，避免为获取媒体库而遍历整个媒体库树。
- 媒体项接口增加 `SearchTerm` 支持，前端搜索和列表加载增加分页加载能力；子文件夹展开时也按页拉取完整子树，避免大目录被固定 `Limit` 截断。
- extras 文件夹缓存增加容量上限，并改为直接调用 Emby `GetExtras` API。
- 统一插件入口和后端 API 的语言归一化逻辑，并将 English (United Kingdom) 作为独立语言代码处理。
- 字幕上传语言选项保留通用 `zh` 中文，使用 `zh-Hans` / `zh-Hant` 表达简体中文和繁体中文，并保留 Emby 可识别的 `zh-HK` 香港繁体标签。
- 统一字幕语言选项的英文显示名称，使用 `Chinese Simplified`、`Chinese Traditional`、`Chinese Traditional (Hong Kong)` 这类 Emby 风格命名。
- 移除前端控制台错误打印和后端上传、删除成功日志，减少不必要的日志噪音。

### 文档

- 安装说明补充插件更新后可能需要清理浏览器缓存或强制刷新 Emby Web 页面。

## v1.3.0 - 2026-05-08

- 加固字幕删除接口：只允许删除当前媒体项已识别的外部字幕，且路径必须位于该媒体的 Emby 元数据目录内。
- 加固字幕上传接口：后端限制字幕格式、语言代码、空文件和 20MB 文件大小，并拒绝覆盖同名字幕。
- 将删除后的元数据刷新改为异步 `await`，避免同步阻塞。
- 将 extras 虚拟文件夹 ID 从 `string.GetHashCode()` 改为基于 SHA256 的稳定 ID，并为 extras 缓存增加过期清理。
- 优化媒体列表性能：列表接口默认不再为每个媒体项附带字幕流，前端改为选中媒体时按需加载字幕信息。
- 前端上传前增加文件大小、空文件、格式白名单和扩展名一致性校验。
